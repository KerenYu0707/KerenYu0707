Project Description

There is growing interest across the University of Toronto (U of T) community in using data to inform and enhance activities related to the teaching and learning mission of the university. These include pedagogical enhancement and innovation within course and program contexts, academic success support at the program and degree levels, potential tools and resources that may support students in their learning journey, and strategic decision-making for the institution.

Product Journey

Product Journey

Sprint 1: Website Tree Testing

  • Prior to joining the Digital Learning Innovations department, significant research has been done. You can read the paper here. So there were already a website structure that I was going to be working with. But I was interested in knowing whether the website architecture was user friendly or not
  • I end up choosing tree testing using Optimal Workshop. Tree testing allows users to choose where they think they can locate a certain piece of information from the website, and Optimal Workshop can track data like first click, time spent on answering question, success rate etc.
  • We were able to recruit over 20 participants from administrative staffs and faculty staffs from various department at UofT. Using the result from tree testing, we found out that the original website architecture are fairly user friendly with a few exceptions. We made some changes in the labels of our navigation links and moved on to the next stage

Sprint 2: Website Prototype

  • We were using WordPress for the website. But because certain privacy concerns we couldn't share the website link with our users. So I opted in to make the design on Figma.
  • The prototype is fully functioning with clickable buttons. and we could run the test on a artificial computer screen from Figma
  • Note to future self: take advantage of the component feature in Figma, it's life changing

    • Sprint 3: Website Usability Test

  • Once we completed the prototype it was time to do some user testing. We were able to recruit 20 participants from the same user group for the tree testing.
  • Because the functionality of the website were fairly intense, we split the group into 3 cohorts randomly.
  • The study was conducted and recorded via Microsoft Teams call in a 1-on-1 interview format, we gave our participants 3-4 tasks to complete, and each group gets a different set of task so we could test every single functionality without causing fatigue to our user. Each session was around 30 min long and the study were conducted from May 15th to June 9th.
  • A questionnaire was given to our participants after they complete the task to gather additional thoughts and feedbacks
  • Fun fact:Because many of our participants are instructors at UofT, a few of the professors were actually my old CS profs. It was quite the experience to be working with my professor, this had to be the most exciting part of the job.

    • Sprint 4 Synthesizing Findings

    • After we wrap up the testing sessions, I spent sometime going through the recordings and jot down some notes, then created a Miro board to synthesizing the findings.
    • I created a affinity diagram to group my notes and from there, I assigned a theme to each of the group of notes.
    • Using these notes, I was able to create a list of design recommendations and report back to the team for them to continue working on.

    Presentation at the Indiana University Learning Analytics Summit

    • I was also invited to share my work along with my colleagues at the Learning Analytics Summit hosted by Indiana University. Where I shared some of the usability testing method used in our study and how it was able to create a more user friendly product.

    Summary

    Overall, this was a great opportunity for me to showcase my UX design skills. Although not a complete UX journey from idea to product, I still manage to apply most of the core steps in a UX study, including user research, UI design and usability testing. I was also able to bring my perspective as a student and to be able to work with professors at UofT was quite the honor. Knowing the fact that my work, no matter how indirect it was, was contributing towards the success of student is something that I felt related and proud of.

    Keren Yu
